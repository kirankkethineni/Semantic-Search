1 Overview

Semantic-Search is a knowledge-driven plant disease classification system that identifies diseases based on high-level semantic features — the visual patterns, colors, and crop type of the affected leaf — rather than low-level pixel statistics. Classification is performed by querying a structured knowledge base, not by running the image through a class-specific classifier head.

This means adding a new disease class requires only a plain-text description. No new labeled images, no retraining. The system extracts entities from the text using NLP, writes a record to the database, and the next inference can immediately recognize it.

🌿
Core Insight Every plant leaf disease can be described as a combination of a semantic shape feature (spots, stripes, patches…) and a set of colors. This structured representation reduces the feature space dramatically and makes the model class-agnostic.

✖ Conventional CNN Approach

  • Trains a separate output neuron per disease class
  • Requires thousands of labeled images per new class
  • Full model retraining to add any new disease
  • Black-box decision — no explanation for output
  • Struggles with intra-class variability across regions/seasons

✔ Semantic-Search Approach

  • CNN detects semantic features, not disease labels
  • New classes added via a single-sentence text description
  • Zero retraining — knowledge base updated with SQL INSERT
  • Explainable: returns the matched semantics and colors
  • Handles intra-class variability through semantic matching

2 Proposed Approach

The system consists of three components working in two distinct workflows — Learning (adding a new disease) and Inference (classifying a leaf image).

3 Technical Components

3.1   CNN Classification Model

The classification CNN predicts which semantic shape category is present on the leaf — it does not predict the disease name directly. There are 7 output classes, each corresponding to a visual pattern observable on diseased leaves.

Spots
Circular/irregular discolorations
Stripes
Linear streaks of different color
Flecks
Tiny scattered specks
Patches
Irregular blotch areas
Powdery
Dust-like white/gray coating
Mosaic
Mixed normal and discolored regions
Velvety
Soft, thick, plush-like texture
Visual examples of the 7 semantic categories used by the classification CNN.

The model is built entirely with SeparableConv2D layers, which factorize the convolution into depthwise + pointwise operations, making the model lightweight (87K parameters) while maintaining accuracy.

InputLayer(None, 256, 256, 3)0

SeparableConv2D (32→128, ×5 blocks)(None, 128→2, 128→2, 128)~68K
GlobalAveragePooling2D(None, 128)0
Dense (32, ReLU)(None, 32)4,128
Dense (7, softmax)(None, 7)231
Total Trainable Parameters87,207

Optimizer: Adam  ·  Loss: sparse_categorical_crossentropy  ·  Input: 256×256 RGB  ·  Validation accuracy: 95.28%

Semantic_Classification.ipynb
# Key model definition — SeparableConv2D block pattern
model = Sequential([
    Conv2D(32, (3,3), activation='relu', input_shape=(256,256,3)),
    MaxPooling2D(),
    SeparableConv2D(32,  (3,3), activation='relu'), MaxPooling2D(),
    SeparableConv2D(64,  (3,3), activation='relu'), MaxPooling2D(),
    SeparableConv2D(128, (3,3), activation='relu'), MaxPooling2D(),
    # ... additional blocks with 128 filters ...
    GlobalAveragePooling2D(),
    Dense(32, activation='relu'),
    Dense(7,  activation='softmax')   # 7 semantic classes
])
# Total: 87,207 trainable parameters

3.2   CNN Segmentation Model

When the classification CNN predicts spots or stripes — features that are spatially localized — a segmentation model is run to precisely locate the diseased regions on the leaf. This localization is necessary because the color of the diseased area must be extracted separately from the background/healthy leaf color.

The segmentation model is a fully-convolutional U-Net-style architecture with skip connections (Conv2DTranspose upsampling + Concatenate) and 1.45 million trainable parameters. It was trained on 3,000 images from a separate leaf segmentation dataset and achieves 93% accuracy and 0.73 mean IoU.

💡
Why only spots and stripes? Powdery, velvety, and mosaic features cover the entire leaf surface and do not require spatial localization — the entire leaf is the diseased region. Only spot and stripe patterns need the segmentation step to isolate discrete lesion areas for accurate color extraction.

3.3   NLP / Named Entity Recognition

The NER model is built on spaCy's tok2vec pipeline. It processes a disease description sentence and identifies four entity types, which are then mapped to database fields.

PLANT — Apple, Tomato, Corn… DISEASE — Powdery Mildew, Rust, Blight… COLOR — Yellow, Brown, Dark Green… SEMANTIC — Spots, Stripes, Patches…

The NER model was trained on 15,400 documents (80% train / 20% dev), generated by paraphrasing and augmenting 120 real disease descriptions from PlantVillage using GPT APIs — producing 36,000 total training records. The model achieves 98% F1-score across all entity types.

# spots     → SEMANTIC

4 Knowledge Base & Classification Logic

4.1   Database Schema (ERD)

Disease records are stored in a normalized SQLite database with three tables. Colors and semantics are stored in separate lookup tables and linked via foreign keys to avoid redundancy and support multi-value fields (a disease can have multiple colors).

TableColumnsPurpose
diseasesid, plant_name, disease_name, semantics_idOne row per disease. Links to a single semantic shape.
colorsid, valueLookup table of valid color names.
semanticsid, valueLookup table: Spots, Flecks, Mosaic, Stripes, Curls, Powdery, Velvety.
disease_colorsdisease_id, color_idMany-to-many join: each disease can have multiple associated colors.

4.2   Adding a Record — Algorithm 1

When the NER pipeline extracts entities from a user's text, the following SQL sequence is executed. Spelled-out in the paper as Algorithm 1:

Algorithm 1 — Insert Disease Record

  1. Look up semantics_id for the extracted semantic value (raise error if not found)
  2. INSERT into diseases (plant_name, disease_name, semantics_id) → get disease_id
  3. For each extracted color:
    1. Look up color_id from colors table
    2. INSERT OR IGNORE into disease_colors (disease_id, color_id)
  4. Commit transaction

Note: if "black" or "gray" is mentioned, both are added — since black spots on leaves can appear gray due to leaf coloring.

TAFER1.ipynb — Flask route: /entities (POST)
# After NER extracts: plant="apple", disease="cedar rust",
#   semantic="spots", colors=["brown","yellow"]

# 1. Resolve semantics_id
cur.execute("SELECT id FROM semantics WHERE value = ?", (semantic,))
semantics_id = cur.fetchone()[0]

# 2. Insert disease record
cur.execute("INSERT INTO diseases (plant_name, disease_name, semantics_id) VALUES (?,?,?)",
            (plant, disease, semantics_id))
disease_id = cur.lastrowid

# 3. Link each color
for color in colors_list:
    cur.execute("SELECT id FROM colors WHERE value = ?", (color,))
    color_id = cur.fetchone()[0]
    cur.execute("INSERT OR IGNORE INTO disease_colors VALUES (?,?)", (disease_id, color_id))
conn.commit()

4.3   Weighted Jaccard Similarity — Algorithm 2

During inference, the colors extracted from the image are matched against all disease records with the same plant name and semantic feature. The Weighted Jaccard similarity assigns higher importance to the first detected color (which has the highest pixel count in the segmented region).

Algorithm 2 — Get Best Match for Disease

  1. Query all disease records matching (plant_name, semantics) with their associated colors
  2. Assign weights to the input colors: weighti = max(0.1, 1.0 − 0.2 × i)   (so color rank 0 gets weight 1.0, rank 1 gets 0.8, rank 2 gets 0.6…)
  3. For each candidate disease record:
  4. Compute union = colors_from_image ∪ record_colors
  5. Compute weighted_intersection = Σ weight[color] for all colors in (colors_from_image ∩ record_colors)
  6. score = weighted_intersection / |union|
  7. Return the disease record with the highest score (or "No Disease Found" if no records exist)
TAFER1.ipynb — Weighted Jaccard scoring
# Build weight dict: first color is most important
weight_dict = {}
for i, color in enumerate(colors_list):
    weight_dict[color] = max(0.1, 1.0 - 0.2 * i)

# Score each candidate record
best_score, best_record = -1, None
for record in sql_results:
    record_colors = set(record[4].split(','))
    union = colors_list_set | record_colors
    weighted_intersection = sum(
        weight_dict[c] for c in colors_list_set & record_colors
    )
    score = weighted_intersection / len(union)
    if score > best_score:
        best_score, best_record = score, record

4.4   HSV Color Filters

After segmentation localizes the diseased region, predefined OpenCV HSV range filters detect which colors are present. Colors are ranked by pixel count in the diseased area — the most dominant color appears first and gets the highest similarity weight.

Known Limitations Two main sources of the 10% error: (1) Segmentation errors — including background or healthy leaf in the segmented region adds spurious colors. 8 Comparison with Prior Work

The table below compares Semantic-Search against four families of prior approaches, all re-implemented and evaluated under identical experimental conditions on the same dataset.

Approach Semantic Understanding Retraining for New Classes Scalability Retraining Cost Accuracy
CNN-based modelsLimited (pixel-level)RequiredLowHigh97%
Histogram-based SVMNoRequiredLowModerate80%
Bag of Visual WordsNoRequiredLowModerate82%
Knowledge GraphYesNo, but complex graph constructionModerateLow83%
Semantic-Search (ours) Yes — Object + Color + NLP No — database update only High Very Low 90%
🔎
The accuracy trade-off is intentional. The CNN baseline achieves 97% because it is trained end-to-end on disease labels — it memorizes class-specific pixel patterns. Semantic-Search sacrifices 7 percentage points to gain: zero retraining, transparent decisions, and the ability to add new disease classes with a single sentence. For precision-agriculture deployments where disease diversity is regional and evolving, this trade-off is well-justified.
